1. Home/Query page [Go to top]

The home page of bioTextQuest is the page where the user can input his query:



Query: The keywords with which the search will be performed over PubMed database. The query should be in the format that is normally accepted by the database and will be processed exactly as it would be processed if the search was performed from the original web site. Any notation that would be acceptable in the original database interface is acceptable here, e.g. p53 AND (leaver OR pancr*) NOT cancer (Medline). The query must return more than 120 articles in PubMed for proper results. In other cases an error is possible.
It should be noted that the abstracts are kept in a local database that is updated every week with new records from PubMed. Thus, some searches may not yield the same number of articles when performed in the bioTextQuest server and the PubMed web site.
Advanced Parameters: Control of stemming and clustering algorithms and their parameters. Upon Clicking on the Advanced button, the user can change the default options:



Choose extract entities: Select which extract entities will be present in the analysis.For example, the user can choose to only inlude genes/proteins from the papers that PubMed returns. If left blank, all entities are included.
Similarity matrix:Clustering can be performed according to various similarity methods such as: Cosine, Jaccard, Euclidean similarity.
Clustering Algorithm: (Default: Louvain community detection). Clustering can be performed by various algorithms. Currently the following clustering algorithms are supported: Louvain community detection, Top2Vec, K-means, Markov Clustering (MCL).
Each Clustering algorithm has parameters whose optimum values (according to the of the author of the implementation) are chosen as default. The user can change them according to his needs using the provided inputs.

  • Louvain community detection, parameter: Resolution (default: 1)
    Resolution is a parameter that affects the size of the recovered clusters. Smaller resolutions recover smaller clusters and therefore a larger number of them, while, conversely, larger values recover clusters containing more data points. More information here
  • Markov Clustering (MCL), parameter: Inflation (default: 1.8)
    The rule of thumb is that the larger the inflation parameter the more clusters will be created and the more fine-grained they will be. More information here
  • K-Means, parameter: number of clusters (default: 3)
    The user specifies the desired number of resulting clusters. More information here
  • Top2Vec
    Implementation of the python algorithm. Needs a minimum number, usually more than 100, of articles to produce results. More information here
Total papers: The maximum number of papers that should be analyzed. If there are more papers than the limit imposed by the user, the more recent ones are retrieved.

2. Results tab [Go to top]

The Results tab contains 3 tabs: Clustered Documents, Wordclouds, Biomedical terms

Clustered documents
This tab displays all the PubMed articles that have been analysed. It includes the cluster that the article has been assigned, the PubMed id with a link to PubMed and the abstract
The Summary shows the total number of analysed papers and the number of generated clusters

A specific seed is used, as recommended in the documentation, in order to obtain reproducible results.


Wordclouds The created wordclouds. Each cluster is represented by keywords, which are displayed as a tag cloud. The size of each term is proportional to its frequency. The colour of the term corresponds to one of the following Extract entities :
  • Genes/Proteins of a specific organism
  • PubChem Compound identifiers
  • NCBI Taxonomy entries
  • Gene Ontology biological process terms
  • Gene Ontology cellular component terms
  • Gene Ontology molecular function terms
  • BRENDA Tissue Ontology terms
  • Disease Ontology terms
  • Environment Ontology terms
Wordcloud Features
All the wordclouds have the following features:
  • Interactive words: When the user clicks the word, a new tab opens where more information regarding this word can be found. Each entity type links to a different database (PubChem, NCBI etc.)
  • Additional options: Using this menu, the user may choose to perform the following actions in all clusters:


    1. Total words in cloud
      Change the total number of words in each wordcloud
    2. Download wordclouds
      Download a PDF copy of the wordclouds
    3. Colour only specific entities
      The user can select which entities will be coloured in the wordclouds by checking the relevant checkbox. The rest will be grey.
    4. Highlight words
      By moving the mouse over an entity in the menu, all the relevant words are highlighted.
Biomedical terms
This tab shows the frequency of each word in its cluster. The words can be sorted in ascending or descending order.
